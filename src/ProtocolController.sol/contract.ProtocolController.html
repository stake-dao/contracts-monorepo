ProtocolController

Git Source

Inherits: IProtocolController, Ownable2Step

Author: Stake DAO

Central registry for protocol components and permissions management

Manages protocol components, permissions, and shutdown functionality

State Variables

gauge

Maps each gauge address to its configuration

This is the primary registry that links external gauges to our vault system

mapping(address => Gauge) public gauge;

registrar

Authorized addresses that can register new vaults and set allocation targets

Typically factory contracts that need to create new vaults programmatically

mapping(address => bool) public registrar;

permissionSetters

Addresses authorized to manage granular function-level permissions

Enables delegation of permission management without giving full ownership

mapping(address => bool) public permissionSetters;

_protocolComponents

Core components for each protocol, indexed by protocol ID

Protocol ID is typically keccak256("PROTOCOL_NAME") truncated to bytes4

mapping(bytes4 => ProtocolComponents) internal _protocolComponents;

_isValidAllocationTargets

Whitelisted allocation targets for each gauge

Strategies can only allocate funds to these pre-approved destinations (e.g., locker, sidecars)

mapping(address => mapping(address => bool)) internal _isValidAllocationTargets;

_permissions

Granular permission system: contract -> caller -> function -> allowed

Enables fine-grained access control for specific function calls

mapping(address => mapping(address => mapping(bytes4 => bool))) internal _permissions;

isPaused

Per-protocol deposit pause state

When true for a protocol, deposits are paused but withdrawals remain functional

mapping(bytes4 => bool) public isPaused;

Functions

onlyRegistrar

Ensures only authorized registrars or owner can register vaults

Used by factory contracts during vault deployment

modifier onlyRegistrar();

onlyStrategy

Ensures only the protocol's strategy can call gauge-specific functions

Prevents unauthorized contracts from marking gauges as withdrawn

modifier onlyStrategy(address _gauge);

onlyPermissionSetter

Ensures only authorized permission setters can modify permissions

Allows delegation of permission management without full ownership

modifier onlyPermissionSetter();

Structs

ProtocolComponents

Stores the core components for each protocol integration

Each protocol (Curve, Balancer, etc.) has its own set of components

struct ProtocolComponents {
    address strategy;
    address allocator;
    address accountant;
    address feeReceiver;
}

Gauge

Links a gauge to its associated vault and protocol

A gauge is the external yield source (e.g., Curve gauge) that the vault interacts with

struct Gauge {
    address vault;
    address asset;
    address rewardReceiver;
    bytes4 protocolId;
    bool isShutdown;
}
